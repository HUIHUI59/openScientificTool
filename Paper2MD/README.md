# Paper2MD · 论文 PDF 转 Markdown

> 把 PDF 论文转成 Markdown/纯文本——喂 AI、进 Obsidian/Notion 做笔记、全文检索都需要这一步。
> **双击 exe 就能转, 不用配任何 Python 环境。** 保留标题层级和段落, 表格尽力还原, 图片可选提取。

```
paper.pdf          ->  paper.md                    (单文件: 同目录同名输出)
我的论文文件夹\     ->  我的论文文件夹\md_output\   (文件夹: 递归批量, 原文件绝不动)
```

---

## 🔴 铁律: 它只转格式, 不动脑子

**这四条既是产品定位, 也是合规边界, 写死在代码里:**

1. **不总结、不改写、不翻译、不润色、不删减**——原文有什么就转出什么, 内容一字不动。
   它是纯格式转换器, 不生成任何学术观点文本。
2. **不做 OCR**(Optical Character Recognition, 光学字符识别, 即把图片里的字认成文本):
   扫描版/图片型 PDF 没有文本层, 程序检测到会如实提示「该文件无文本层, 本工具不支持」,
   绝不硬转出一堆乱码骗你。
3. **不内嵌任何 AI 模型、不调任何 AI API**——它是解析器, 不是助手。全程本地离线运行,
   不联网、不上传任何文件, 论文稿件不出你的电脑。
4. **不修改、不移动、不删除你的原 PDF**——只读输入, 新建输出。

另外一句常识: **请只转换你有权使用的文献文件。**

---

## 这是什么 / 给谁用

想把 PDF 论文喂给 AI 工具(Claude / GPT / 本地知识库), 或想在 Obsidian / Notion 里
做全文笔记检索, 第一步都是「PDF → 可编辑文本」。直接从 PDF 复制粘贴的结果大家都见过:
断行乱序、表格糊成一团。

**Paper2MD 专治这一步。** 它把 PDF 转成规整的 Markdown:
标题层级(`#`/`##`/`###`)、段落、列表照排; 简单表格转成 Markdown 表格;
图片可选提取到子目录并在 `.md` 里用相对路径引用。

- **单文件模式**: 拖入一个 PDF → 同目录输出同名 `.md`。
- **批量模式**: 拖入一个文件夹 → 递归找出所有 `.pdf` 逐个转换, 输出到 `md_output/` 子目录。

> 与重型解析工具(如 MinerU)的关系: 它们解析力更强(公式/复杂版面), 装得上环境很值得用;
> Paper2MD 是给**装不上环境的人**的轻量选项——免模型下载、免 GPU、免任何配置, 双击就能跑。
> 互补, 不对抗。
>
> 本仓库另有 [`skills/paper-digest`](../skills/paper-digest/) 面向 Claude Code 用户直接读 PDF;
> Paper2MD 服务的是不用 Claude Code 的更广人群, 两者互不依赖。

## 用法①: 双击运行(最省事, 首推, 不用装 Python)🖱️

1. 到本仓库 **Releases** 页面下载 `Paper2MD.exe`(单文件, 无需安装任何环境)。
2. **双击 `Paper2MD.exe`**, 弹出一个黑色小窗口, 提示:
   > 把 PDF 文件或文件夹拖到本窗口再按回车, 或直接粘贴路径:
3. 把**单个 PDF** 或**整个文件夹**拖进窗口, 按回车。
4. 它会问一句「是否提取图片?(y/N)」——要图就按 `y`, 纯文字直接回车。
5. 逐文件打印进度(`[2/5] xxx.pdf → done`), 结尾汇总输出位置。
6. 窗口停在「按回车键退出……」, 看完结果再关——**不会一闪而过**。

---

## 用法②: 把 PDF/文件夹拖到 exe 图标上

懒得双击再拖窗口的话, **直接把 PDF 文件或文件夹拖到 `Paper2MD.exe` 的图标上松手**,
效果和用法①一样: 询问是否提取图片 → 转换 → 汇总 → 结束暂停。

## ⚠️ 首次运行会弹蓝色「Windows 已保护你的电脑」——这是正常的

因为这个 exe 是个人做的、**没有花钱买微软代码签名证书**, Windows 的 SmartScreen 第一次
遇到它会谨慎地弹一个蓝色窗口「Windows 已保护你的电脑 / Windows protected your PC」。

**这不是病毒报警, 是"没签名"提醒。** 处理办法:

1. 点窗口里的「**更多信息 / More info**」(一开始是折叠的)。
2. 出现「**仍要运行 / Run anyway**」按钮, 点它即可。

之后再运行就不会再弹了。(介意的话可以用下面「从源码运行」的方式, 自己看代码更放心。)

> 提示: exe 第一次启动会慢几秒(它要把自己解压到临时目录), 属正常现象。

## 用法③: 命令行运行(会用命令行的人)

```bat
:: 单个 PDF -> 同目录同名 .md(会交互询问是否提取图片)
Paper2MD.exe "C:\论文\paper.pdf"

:: 整个文件夹批量转换, 并提取图片(不再询问)
Paper2MD.exe "C:\我的\论文文件夹" --images

:: 批量转换、不提取图片、结束不暂停(给脚本/批处理用)
Paper2MD.exe "C:\我的\论文文件夹" --no-images --no-pause
```

- `--images`: 提取图片到 `<文件名>_images/` 子目录, `.md` 内用相对路径引用。
- `--no-images`: 不提取图片, 且不再交互询问。
- `--no-pause`: 结束不暂停(被别的脚本调用时用)。

## 用法④: 从源码运行(会用 Python 的人)

```bash
pip install pymupdf4llm      # 唯一必需依赖, 免模型下载、免 GPU、离线可用
python paper2md.py "paper.pdf"                       # 单文件
python paper2md.py "你的文件夹" --images             # 批量 + 提取图片
```

- Python 3.9+ 即可。
- 依赖见 `requirements.txt`(核心就是 `pymupdf4llm`, PyMuPDF 官方出品的 PDF→Markdown 库)。

## 打包成 exe(想自己打包的人)

**必须用干净的 venv**, 别在 Anaconda 环境里直接打——Anaconda 会把无关库全卷进去,
exe 体积能从 ~44 MB 暴涨到 200 MB 以上(本仓库 PaperTidy 踩过的实坑)。

```bat
:: 1. 建干净虚拟环境
python -m venv venv_build
venv_build\Scripts\activate

:: 2. 只装必需的东西
pip install pymupdf4llm pyinstaller

:: 3. 单文件打包(必须加 --collect-data pymupdf, 原因见下方坑)
pyinstaller --onefile --name Paper2MD --collect-data pymupdf paper2md.py

:: 产物在 dist\Paper2MD.exe
```

> ⚠️ **打包坑(07-10 实测)**: 一定要加 `--collect-data pymupdf`。PyMuPDF 内部带一批数据资源
> (含 onnx 等文件), PyInstaller 默认不会把它们收进 exe, 漏了打出来的 exe 一运行就报资源缺失错误。
> 加上这个参数, 打包器才会把 pymupdf 的数据文件一并打进去。

## 注意事项(边界如实说)

- **扫描件不支持**: 无文本层的扫描版/图片型 PDF, 程序会如实提示并跳过——本工具不做 OCR,
  不会硬转出乱码。需要识别扫描件请另寻带 OCR 功能的工具。
- **复杂表格尽力还原**: 简单表格可以转成 Markdown 表格; 跨页表、合并单元格的表
  还原效果有限, **重要数据请对照原文人工核对**——不装完美。
- **双栏排版**: 学术论文常见的双栏版式, 阅读顺序绝大多数情况能正确还原, 但个别版式
  复杂的页面可能有段落顺序问题, 同样建议关键处对照原文。
- **输出 = 原文的如实转换**: 转出来的 `.md` 该是什么就是什么, 后续怎么用(喂 AI、做笔记)
  由你决定并由你负责。
- 全程离线, 不联网。转换耗时与文件大小和页数相关, 现场如实打印, 无任何速度承诺。
