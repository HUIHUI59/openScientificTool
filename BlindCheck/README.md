# BlindCheck · 匿名处理自查器

> 双盲投稿、学位论文盲审送审前，把 docx / PDF 拖进来，
> 扫出所有**会暴露你身份的地方**——输出「泄露清单 + 手动修复步骤」。
> **只读扫描，不改你的文件；纯本地运行，零联网。**

```
 1. 【高危】 docx文件属性·作者          -> Zhang Wei
 3. 【高危】 修订记录署名              -> Zhang Wei
 4. 【高危】 批注署名                  -> 张伟
 5. 【留意】 自引式措辞  正文第 2 段    -> …笔者此前的研究（张伟，2023）表明…
 7. 【留意】 基金号/项目编号 正文第 3 段 -> …国家社会科学基金项目（21BSH123）…
```

（上面是本工具在一份埋了 6 类泄露的测试稿上跑出来的真实结果，6 类全部检出。）

---

## 🔒 四条铁律（先读这里，写死在代码里）

1. **只读扫描，绝不修改你的文件。** 修复由你按清单里的步骤自己动手——自动改稿有损坏文件的风险，这个工具不冒。
2. **只输出清单，不生成任何文本。** 它不会生成匿名化后的替代句；它的输出永远不能直接粘进论文。
3. **不承诺安全。** 覆盖的是六类**常见**泄露，非穷尽；检出为 0 ≠ 匿名处理绝对完备，投出前请再人工过一遍。
4. **零联网、零上传。** 纯本地扫描，你的稿子不出这台电脑（代码开源，可自行核查确实没有任何网络请求）。

---

## 这是什么 / 给谁用

投双盲期刊、送学位论文盲审，都要求稿件里看不出作者是谁。正文里的名字你会删，
但**藏在文件里的身份信息**大多数人不知道：Word 文件属性里写着你的名字、
开着修订模式改过的稿子每处修订都带署名、导师批注带署名、导出的 PDF 元数据里还有一份、
致谢里写着导师名、基金号能直接查到你的立项信息、"笔者此前的研究"一句话就把自引暴露了。

**BlindCheck 专治这个。** 它逐项扫这六类，告诉你泄露在哪、怎么手动修。

适合：投双盲期刊的社科 / 人文 / 理工科研究者、学位论文送审前做匿名处理的硕博生。
**不需要你会写代码。**

## 六类检查项

| # | 检查项 | 查哪里 | 等级 |
|---|---|---|---|
| ① | docx 文件属性 | 作者 / 最后修改者（`docProps/core.xml`） | 高危 |
| ② | 修订记录署名 | 开着修订模式留下的每处署名（`document.xml`） | 高危 |
| ③ | 批注署名 | 批注作者（`comments.xml`） | 高危 |
| ④ | PDF 元数据 | `/Author` `/Creator` 等字段 | 高危 |
| ⑤ | 致谢与基金号 | 致谢段落、基金/项目编号模式（中英双语正则） | 留意 |
| ⑥ | 自引式措辞 | "笔者此前的研究 / in our previous work / 见拙作"类（中英双语正则） | 留意 |

每一项检出都附**手动修复步骤**（如：Word → 文件 → 信息 → 检查问题 → 检查文档 → 删除个人信息）。
检出为 0 时也会列出"已检查的六类"，让你知道它查过什么。

## 用法①：双击运行（最省事，首推，不用装 Python）🖱️

1. 到本仓库 **Releases** 页面下载 `BlindCheck.exe`（单文件，无需安装任何环境）。
2. **双击 `BlindCheck.exe`**，黑色小窗口会先亮出四条铁律，然后提示：
   > 把要检查的 docx/PDF 拖到本窗口再按回车
3. 把你的稿子文件拖进窗口，回车，泄露清单直接列出来。
4. 可以连着查多个文件（比如 docx 一份、导出的 PDF 一份，各查一遍）；直接回车退出。
5. 结束后窗口会停住等你按回车——**不会一闪而过**。

## 用法②：把文件拖到 exe 图标上

把 docx / PDF **直接拖到 `BlindCheck.exe` 的图标上松手**，立即出结果，结尾同样暂停等你看完。

## ⚠️ 首次运行会弹蓝色「Windows 已保护你的电脑」——这是正常的

这个 exe 是个人做的、没有花钱买微软代码签名证书，SmartScreen 第一次遇到会谨慎提醒。
**这不是病毒报警，是"没签名"提醒。** 点「**更多信息 / More info**」→「**仍要运行 / Run anyway**」即可。
介意的话，代码全开源，可以用下面「从源码运行」的方式自己看着代码跑。

## 用法③：命令行

```bat
BlindCheck.exe "C:\我的\投稿匿名版.docx" "C:\我的\投稿匿名版.pdf"
BlindCheck.exe 稿子.docx --no-pause   :: 被脚本调用时结尾不暂停
```

## 用法④：从源码运行（会用 Python 的人）

```bash
pip install pypdf                 # 唯一依赖, 只用于读 PDF; 只查 docx 可不装
python blind_check.py 稿子.docx 稿子.pdf
```

Python 3.8+ 即可，其余全是标准库。

## 原理（为什么它可靠、为什么它安全）

- docx 本质是个 zip 包：用标准库 `zipfile` + 正则**直读内部 XML**（`core.xml` 属性、
  `document.xml` 修订署名与正文、`comments.xml` 批注），不依赖第三方解析器，更稳。
- PDF 元数据与正文用 [`pypdf`](https://github.com/py-pdf/pypdf) 读取。
- 全程只有"读"操作，**代码里不存在任何写文件、改文件的调用**；也不存在任何网络请求。
- 单项检查失败会降级为"该项无法检查"照常输出其余项，不会整体崩掉。

## 注意事项

- **六类非穷尽**：图片里的水印、页眉页脚、参考文献列表里的自引密度等不在覆盖范围，仍需人工过一遍。
- ⑤⑥ 两类靠正则模式匹配，可能有误报/漏报，报出来的是"值得你看一眼的位置"，取舍你来判断。
- 老格式 `.doc` 不支持，请先用 Word 另存为 `.docx`；加密文档会提示"无法检查"而不是硬崩。
- PDF 正文抽取效果取决于 PDF 本身（扫描版抽不出文字），抽不出时建议回到 docx 源文件上查。
