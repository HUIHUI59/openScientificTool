# 任务表输出模板（可直接复制）

> review-decoder 的标准输出格式。四块缺一不可，尤其是**固定尾注两句**。占位符用 `<>` 标出。

```markdown
# 审稿意见拆解清单 · <稿件代称，用户没给就留空>

> 拆解时间：<YYYY-MM-DD> ｜ 意见来源：<期刊审稿 / 学位论文盲审 / 外审>
> 🔒 本清单只做拆解整理，不含任何可直接提交的回复文字。

## 一、总览

- 审稿人：<N> 位（<R1 / R2 / 编辑…>）；拆出原子任务 <M> 条。
- 类别 × 优先级矩阵：

| | P0 不改就拒 | P1 重要 | P2 锦上添花 | 小计 |
|---|---|---|---|---|
| 数据 | <n> | <n> | <n> | <n> |
| 方法 | <n> | <n> | <n> | <n> |
| 写作 | <n> | <n> | <n> | <n> |
| 补充说明 | <n> | <n> | <n> | <n> |
| **合计** | <n> | <n> | <n> | **<M>** |

- ⚠️ 特别注意：<多位审稿人重合的问题 / 互相冲突的意见，没有就写「无」>

## 二、任务清单（按 P0 → P1 → P2 排）

| 勾 | 编号 | 优先级 | 类别 | 工作量(预估) | 任务（一句话动作） | 备注 |
|---|---|---|---|---|---|---|
| [ ] | R1-01 | P0 | 数据 | 周级 | <做什么，一个动作> | <如：R1/R2 同提> |
| [ ] | R1-02 | P0 | 方法 | 天级 | <…> | |
| [ ] | R2-01 | P1 | 写作 | 小时级 | <…> | |

### 原文锚点（编号对照，一字不差引用）

- **R1-01** ｜ 原文：
  > <审稿人原句，一字不差；英文意见可另起一行附中文对照翻译，但原文必须保留>
- **R1-02** ｜ 原文：
  > <…>

## 三、依赖关系（没有可整块省略）

- R1-01 补数据 → R1-02 重跑回归 → R2-03 更新结果小节
- <其余链条…>

## 四、尾注（固定两句，不可省略）

- 以上分类／优先级／工作量为拆解建议，请与导师或合作者确认后再执行。
- 本清单只做拆解整理，不含任何可直接提交的回复文字——回复怎么写，由你自己完成。
```

## 使用要点

1. **编号规则**：`R<审稿人序号>-<条目序号>`，编辑意见用 `ED-<序号>`。原意见的第几条不必对应——一条原意见拆出的多个原子任务各占一个编号，靠锚点回溯。
2. **任务列只写动作**：「补报量表的 Cronbach's α」而不是「审稿人认为信度信息不足」；动词开头。
3. **锚点独立成块**：表格里塞长引用会挤爆列宽，所以锚点单独列在表后，编号对照。
4. **排序**：先优先级（P0 → P2），同级内按依赖先后（被依赖的排前面），再按审稿人序号。
