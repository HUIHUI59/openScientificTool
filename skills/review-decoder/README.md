# review-decoder · 审稿／盲审意见拆解 skill

> 把收到的审稿意见／盲审意见拆解成**逐条修改任务清单**：按数据／方法／写作／补充说明分类，标 P0-P2 优先级和工作量档位。
> **只拆解整理，绝不替你写任何回复文字**——这条焊死在 skill 里。

---

## 这是什么 / 给谁用

收到审稿意见或盲审意见的那一刻：满屏英文、成段散文、一条里藏着三个要求，不知道从哪改起、哪条不改会致命。

**review-decoder 专治这个。** 把意见全文粘给装了它的 AI，它会：

1. **切原子条目**——一条长意见拆成一个个独立动作（一段话里藏的 3 个要求全拆出来）；
2. **四类分类**——数据（补数据／重新分析）／方法（方法论质疑）／写作（表述／结构）／补充说明（需解释论证）；
3. **标优先级与工作量**——P0 不改就拒／P1 重要／P2 锦上添花 ＋ 小时级／天级／周级 ＋ 哪条要先做；
4. **产出可勾选的 Markdown 任务表**——每条带一字不差的原文锚点，方便对照。

适合：收到期刊审稿意见、学位论文盲审／外审意见的硕博与科研人员。**不需要你会写代码。**

## 🔒 它绝不做什么（先说清楚）

- **绝不写回复信（response letter）、申辩文字、修改说明**——一个字的「可提交文字」都不产出。为什么不直接丢给 GPT？裸问它会顺手替你把回复也写了——那是你的学术表达，必须你自己来。本 skill 把「只拆不写」写进铁律，每次自动生效。
- 不评价审稿人对错、不教你反驳、不预测录用概率。
- 分类和优先级只是建议，任务表自带提醒：**请与导师／合作者确认后再执行**。

## 用法①：有 AI agent（Claude Code / Codex 等）🔌

把下面这句话原样发给你的 agent：

> 请安装一个 GitHub skill：仓库是 HUIHUI59/openScientificTool，skill 在 skills/review-decoder 目录。把该目录复制到你的 skills 目录（如 `~/.claude/skills/review-decoder`），装好后我新开窗口使用。

装好后这样用：

> 用 review-decoder 拆解这份审稿意见：【粘贴意见全文】

## 用法②：没有 agent，任意网页 AI 也行 💬

把 [`SKILL.md`](./SKILL.md) **全文复制**，粘贴给 ChatGPT／Claude／豆包等任意对话框，再补一句：

> 用以上规则拆解下面这份审稿意见：【粘贴意见全文】

效果一致——skill 本质就是一份带规矩的说明书。意见特别长（3 页以上）时，**按审稿人分批喂**更细。

## 能得到什么效果

输入一大段密密麻麻的意见，输出一份任务表：总览矩阵（类别 × 优先级数量）→ 可勾选 checklist（编号／优先级／类别／工作量／动作）→ 原文锚点对照 → 依赖链（先补数据 → 再重跑 → 再改稿）。

完整的「输入 → 输出」对照样例见 [`examples/sample-review.md`](./examples/sample-review.md)（模拟语料，不对应任何真实论文）——里面能看到一条长意见被拆成 3 个原子任务的实际效果。
