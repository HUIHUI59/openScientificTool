# cite-check · 参考文献打假体检 skill

> 把参考文献清单交给你的 AI，逐条拿 **CrossRef 官方数据库**核验：DOI 是真是假、
> 题目/作者/年份对不对得上、格式统不统一、缺了哪些字段——出一份 ✅/⚠️/🔴 体检报告。
> **只体检已有条目，绝不代找文献。** 这是一份给 AI 的「带规矩的说明书」，复制即装。

---

## 这是什么 / 给谁用

AI 编造不存在的文献（幻觉引用）是老问题了：题目像模像样、作者真实存在、DOI 格式也对——但那篇论文根本不存在。混进参考文献列表里，投稿前不查出来就很难看。

**为什么不直接问 GPT"这条文献是真的吗"？** 因为它自己就是会编文献的那位，它说"是真的"你也不敢信。
**cite-check 让 AI 去问 CrossRef 官方数据库**（全球学术出版物的 DOI 注册机构）：404 就是 404，查得到就是查得到。报告里的每个 DOI 你都可以自己去 doi.org 再验一遍，不必信 AI 的一面之词。

适合：所有要交参考文献列表的人——投稿前、送审前、把 AI 起草的文献清单收进论文之前。

## 它守的四条铁律（写死在 SKILL.md 最前面）

1. **绝不代找文献**——不接"帮我的观点配几条引用"这种活；
2. **绝不输出可粘进论文的内容**——产出只有体检报告和修复建议；
3. **绝不自动改写你的条目**——错在哪、官方值是什么它告诉你，改由你自己动手；
4. **绝不对"无法核验"下真伪结论**——中文知网/万方文献大多不在 CrossRef，它会如实说"核不了"并告诉你去哪自查，不装能。

## 用法①：有 AI agent（Claude Code / Codex 等）🔌

把下面这句话原样发给你的 agent：

> 帮我安装 GitHub 上 HUIHUI59/openScientificTool 仓库里 skills/cite-check 这个 skill：把该目录复制到你的 skills 目录（如 `~/.claude/skills/cite-check`），装好后我新开窗口使用。

装好后这样用（`.bib` 文件或整段参考文献文本都行）：

> 用 cite-check 体检下面这份参考文献：【粘贴你的参考文献清单】

agent 会自己解析条目（乱格式的粘贴文本也能拆）、逐条调 CrossRef 核验（自带礼貌间隔），最后给你逐条 ✅/⚠️/🔴 的报告。

## 用法②：没有 agent，任意网页 AI 也行 💬

把 [`SKILL.md`](./SKILL.md) **全文复制**粘贴给 ChatGPT / Claude / 豆包等对话框，再补一句：

> 用以上规则体检下面这份参考文献：【粘贴清单】

注意：网页 AI 若不能联网调 API，只能做**格式与缺项体检**；真伪体检需要能跑代码或联网的环境。都没有的话，用同仓库的 [`CiteCheck`](../../CiteCheck) 双击版 exe，效果一样。

## 能得到什么

- **逐条体检**：每条 = 状态（✅通过 / ⚠️待核 / 🔴高危）+ 核验方式 + 发现 + 修复建议（改哪个字段、CrossRef 官方值是什么）；
- **清单级体检**：全半角标点混用、作者写法混用、DOI 写法混用、缺卷期页码等；
- **诚实的边界**：中文文献核不了真伪就明说，并给"知网/万方按题名自查"的替代动作。

```text
cite-check/

```

## 理念(HITL, 人在回路)

它只负责**核对与提醒**——找出可疑条目、给出官方值；判断和修改永远是你自己的。
报告不是论文内容，改完请再自己过目一遍。
