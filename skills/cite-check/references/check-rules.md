# cite-check · 体检项清单与判定标准（细则）

> SKILL.md 是总纲；本文件是逐项判定标准，体检时照这里的阈值执行。

## 1. 真伪体检

### 1.1 有 DOI 的条目（DOI 直查）

接口：`GET https://api.crossref.org/works/{doi}`（带礼貌池 User-Agent，间隔 ≥1 秒）。

| 返回 | 判定 |
|---|---|
| `404` | 🔴 高危：DOI 查无此号，疑似编造或抄写错误。建议：先自己到 doi.org 粘贴复核一次；确认不存在就回原始出处找正确 DOI（不代找）。 |
| `200` | 进入 1.3 字段比对 |
| 其他 / 超时 | ⚠️ 服务暂不可用，本条未核验（≠有问题），建议稍后重跑 |

### 1.2 无 DOI 的条目（模糊检索）

接口：`GET https://api.crossref.org/works?query.bibliographic={题目+作者+年份}&rows=3`，取题目相似度最高的一条为候选。

| 情形 | 判定 |
|---|---|
| 候选题目相似度 ≥0.85 且年份不冲突 | ✅ 通过：给出候选题目 + DOI，提醒用户自行到 doi.org 复核后把 DOI 补进条目（用户动手） |
| 候选相似度 ≥0.85 但年份冲突 | ⚠️ 待核：列出出入，让用户确认是否同一篇 |
| 无高相似候选 且 条目含中文 | ⚠️ 无法核验：中文文献大多不在 CrossRef；建议到知网/万方按题名自查。**绝不判假** |
| 无高相似候选（外文） | ⚠️ 无法核验：图书/报告/未收录期刊都可能查不到；建议 Google Scholar / 出版社官网按题名自查。**绝不判假** |

### 1.3 字段比对（DOI 返回 200 后）

规范化规则：题目去 `{}`、转小写、去标点、合并空格后算相似度（difflib ratio 或等价）；年份取 CrossRef `published-print → published-online → issued → created` 首个可用值；作者取 `author[].family`。

| 字段 | 一致 | 小出入（⚠️） | 明显不一致（🔴） |
|---|---|---|---|
| 题目 | 相似度 ≥0.85 | 0.5–0.85（副标题/大小写差异，提请逐字核对） | <0.5（DOI 指向另一篇论文） |
| 年份 | 相等 | 差 1 年（在线发表 vs 正式刊出，常见）| 差 ≥2 年 |
| 第一作者 | 官方姓出现在条目中 | 未找到（提请核对作者名单） | ——（作者单独不升 🔴，随题目/年份定） |

任何 🔴 都必须附 CrossRef 官方值 + 一句修复建议（改哪个字段成什么，由用户自己改）。

## 2. 缺项体检（逐条）

缺项一律 ⚠️，不升 🔴。按条目类型查必备字段：

| 类型 | 必备字段 |
|---|---|
| 期刊文章 article | author / title / journal / year / volume / pages |
| 图书 book | author / title / publisher / year |
| 书籍章节 incollection | author / title / booktitle / publisher / year |
| 会议论文 inproceedings | author / title / booktitle / year |
| 学位论文 phdthesis / mastersthesis | author / title / school / year |
| 粘贴纯文本 | 至少要能找到年份；找不到列为缺项 |

## 3. 格式统一体检（整份清单，清单级问题）

- **标点**：同一清单里全角（，。；）与半角（, . ;）混用 → 报数量，建议按投稿刊物统一。
- **作者省略**：`et al.` 与「等」同时出现 → 按投稿语言统一。
- **作者写法**：「姓, 名」式与「名 姓」式混用（.bib 看 author 字段）→ 统一一种。
- **期刊名**：同一期刊出现全称与缩写两种写法 → 必报。
- **DOI 写法**：`https://doi.org/...` 链接式与 `doi:` 前缀式混用 → 统一一种。
- **尾标点**：有的条目句号结尾、有的没有 → 报数量。

## 4. 铁律映射（绝不做）

| 请求 | 回应 |
|---|---|
| 「帮我配几条引用 / 补几篇文献」 | 拒绝：本 skill 只体检已有条目，不代找文献 |
| 「把修好的参考文献列表直接给我」 | 拒绝改写：报告里给官方值和建议，用户自己改 |
| 「查不到就是编的吧？」 | 不下结论：无法核验 ≠ 编造，给自查途径 |
| 「顺便把这段话加上引用」 | 拒绝：不输出任何可粘进论文的内容 |
