# paper-digest · 带页码锚点的文献精读笔记 skill

> 喂一篇论文 PDF，产出一张**六段式精读学习卡**：每条要点都钉着 `[p.7]"原文引句…"` 这样的页码锚点。
> 这是一份给 AI 的「带规矩的说明书」，复制即装、粘贴即用——**只做读懂与整理，笔记仅供你自己复习**。

---

## 这是什么 / 给谁用

读文献是硕博最高频的动作，也最容易半懂不懂：从头硬啃太慢，直接把 PDF 丢给 AI 问"讲了啥"又不敢信——它经常**编页码、编结论**，你还得回去逐句对。

**paper-digest 专治这个。** 它把精读规矩焊死在 [`SKILL.md`](./SKILL.md) 里：

- **六段固定结构**：研究问题／理论框架／数据与方法／核心发现／局限与作者自述不足／中英术语表；
- **每条要点必须带页码锚点＋原文引句**，查不到原句的**不许写**；
- 原文没写的一律标「原文未提及」，不脑补；
- 产出后 AI 还要**自己抽 3 条锚点回原文核对**，核不上的删。

适合：社科／人文／理工科的硕博和科研人员——所有需要精读文献、又想留下可复核笔记的人。**不需要你会写代码。**

## 为什么不直接把 PDF 丢给 AI 裸问

裸问时，AI 的回答没有任何自证机制：它说"第 7 页提到 β=0.21"，你只能选择信或不信。
本 skill 把「每条必须带页码原句、查不到不许写」变成硬约束——**每条笔记都自带"去哪核对"的地址**，
翻到那一页就能验证；AI 交卷前还必须自己抽 3 条核一遍并把结果记在笔记末尾。

## 用法①：有 AI agent（Claude Code / Codex 等）🔌

把下面这句话原样发给你的 agent：

> 帮我安装 GitHub 上 HUIHUI59/openScientificTool 仓库里 skills/paper-digest 这个 skill。把该目录复制到你的 skills 目录（如 `~/.claude/skills/paper-digest`），装好后我新开窗口使用。

装好后这样用：

> 用 paper-digest 精读这篇论文：【PDF 路径】

## 用法②：没有 agent，支持传 PDF 的网页 AI 也行 💬

把 [`SKILL.md`](./SKILL.md) **全文复制**，粘贴给任意支持上传 PDF 的对话框（Claude／ChatGPT 等），再上传论文 PDF，补一句：

> 用以上规则帮我精读附件这篇论文。

注意：网页 AI 的按页定位能力弱于 agent，**锚点页码可能有偏差，用前务必自己抽查几条**。要稳就用 agent。

## 能得到什么

一份 Markdown 笔记卡（真实产出样例见 [`examples/sample-note.md`](./examples/sample-note.md)——对一篇开放获取论文 Klar et al. 2020, PLOS ONE, DOI: 10.1371/journal.pone.0229446 的完整精读卡，38 条锚点全部逐页回读核对通过）：

- 六段条目式要点，每条形如：`核心问题：… —— [p.2]"原文引句…"`；
- 中英术语表：英文原文＋中文＋一句大白话，供定期复习；
- 尾部固定记录 AI 自查结果（抽查了几条锚点、通过几条）。

## 边界（先说清楚，再开始用）🔒

- 它**不合并多篇文献**、不产出"研究现状"式的连贯成段文字、不生成任何能直接粘进论文／作业／投稿材料的段落——输出体裁锁死为条目笔记卡。
- 它**不替你做引用决策**：不会告诉你"这篇适合引在哪一节"。
- 笔记是**你自己的学习卡**：锚点请自己翻页抽查核对；核对通过也只代表"原句在该页"，解读对不对仍要你自己判断。
- 超长文献（>50 页）建议分章喂，一次读不完硬塞会掉质量。

## 目录结构

```text
paper-digest/
├── SKILL.md                    # skill 本体：铁律 + 工作流 + 输出格式规范（粘贴给 AI 的就是它）
├── README.md                   # 本文件 · 说明书
├── references/
│   ├── note-template.md        # 六段笔记模板（字段级定义 + 锚点格式）
│   └── glossary-rules.md       # 中英术语表提取规则
└── examples/
    └── sample-note.md          # 真实产出样例（开放获取论文的完整精读卡，锚点已核对）
```

## 理念（HITL，人在回路）

AI 只负责把论文拆成带地址的要点卡，**读懂、判断、使用永远是你自己的事**。
每条锚点都是一个"欢迎来查我"的邀请——这是本 skill 和裸问 AI 最大的差别。
