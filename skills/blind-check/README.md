# blind-check · 匿名处理自查 skill

> 双盲投稿、学位论文盲审送审前，让 AI 扫一遍你的 docx / PDF，
> 找出所有**会暴露你身份的地方**——只出「泄露位置清单＋手动修复步骤」，
> **绝不改你的文件、绝不替你生成文本**。
> 这是给 AI 的一份「带规矩的说明书」，复制即装、粘贴即用。想要双击就能跑的独立程序，见同仓库 [`BlindCheck/`](../../BlindCheck)（exe 版）。

---

## 这是什么 / 给谁用

匿名送审的稿子，正文里的名字你会删，但大多数人不知道身份还藏在这些地方：
**Word 文件属性**里写着你的名字、开着**修订模式**改过的每一处都带署名、导师**批注**带署名、
导出的 **PDF 元数据**里还有一份、**致谢**里有导师名、**基金号**能查到立项人、
「笔者此前的研究」一句话就把自引暴露了。

直接让 AI「帮我匿名化」，它常顺手替你改稿、编替代句——改坏了没人知道。
**blind-check 把「不许动」焊死**：4 条铁律＋六类检查项＋固定输出格式，写在 [`SKILL.md`](./SKILL.md) 里。
装上之后，AI 只当探测器：报位置、给手动修复步骤，**动手的永远是你自己**。

适合：投双盲期刊的社科／人文／理工科研究者、送审前做匿名处理的硕博生。

## 用法①：有 AI agent（Claude Code / Codex 等）🔌

把下面这句话原样发给你的 agent：

> 请安装一个 GitHub skill：仓库是 HUIHUI59/openScientificTool，skill 在 skills/blind-check 目录。把该目录复制到你的 skills 目录（如 `~/.claude/skills/blind-check`），装好后我新开窗口使用。

装好后这样用：

> 用 blind-check 查一下这份稿子送审前会不会暴露我的身份：【文件路径】

环境能跑 Python 时，agent 会调用 [`scripts/blind_check.py`](./scripts/blind_check.py)（只读扫描脚本，代码可自行核查：无写文件调用、无网络请求），六类一次扫全。

## 用法②：没有 agent，任意网页 AI 也行 💬

把 [`SKILL.md`](./SKILL.md) **全文复制**，粘贴给 ChatGPT／Claude／豆包等任意对话框，再补一句：

> 用以上规则帮我做投稿前的匿名处理自查。

它会走「手工自查」分支：逐项告诉你去 Word／PDF 阅读器的哪里看，你报回所见，它汇总成清单。
（⚠ 别把整篇稿子粘给网页 AI——那等于把未发表稿件上传出去；手工分支只需要你自己看、报结果。）

## 能得到什么效果

固定四块输出：**检查范围 → 泄露清单（等级＋类别＋位置＋命中内容＋手动修复步骤）→ 无法检查的项 → 边界声明**。
下面是脚本在一份埋了 6 类泄露的测试稿上跑出的真实片段（6 类全部检出）：

```
 1. 【高危】 docx文件属性·作者          -> Zhang Wei
 3. 【高危】 修订记录署名              -> Zhang Wei
 5. 【留意】 自引式措辞  正文第 2 段    -> …笔者此前的研究（张伟，2023）表明…
 7. 【留意】 基金号/项目编号 正文第 3 段 -> …国家社会科学基金项目（21BSH123）…
```

想自己复现：跑 [`scripts/make_fixture.py`](./scripts/make_fixture.py)（需 `pip install python-docx pypdf`）生成同款测试稿，再用脚本扫它。

## 目录结构

```text
blind-check/
├── SKILL.md              # skill 本体: 铁律 + 六类检查项 + 工作流 + 输出格式(粘贴给 AI 的就是它)
├── README.md             # 本文件 · 说明书
└── scripts/
    ├── blind_check.py    # 只读扫描脚本(与 exe 版同引擎; 依赖: pypdf, 仅读 PDF 时需要)

```

## 理念（HITL，人在回路）

工具只负责**找**，改永远是你自己改——这不是偷懒，是设计：自动改稿有损坏文件的风险，
而且匿名处理的每一处取舍（致谢怎么放、自引怎么表述）只有作者本人能定。
六类覆盖常见泄露但**非穷尽**，检出为 0 也请在投出前再人工过一遍。
