# PaperTidy · PDF 文献批量改名器

> 把硬盘里一堆 `1706.03762v7.pdf`、`SSRN-id999.pdf`、`download (3).pdf` 这种鬼名字，
> 一键批量改成 **`作者_年份_标题.pdf`**。提不到信息的安全跳过，**绝不乱改**。

```
1-s2.0-S0000000.pdf   ->  He_2015_Deep Residual Learning for Image Recognition.pdf
1706.03762v7.pdf      ->  Vaswani_2017_Attention Is All You Need.pdf
download (3).pdf      ->  Devlin_2018_BERT Pre-training of Deep Bidirectional Transformers...pdf
```

（上面是本工具在真实 PDF 上跑出来的实际结果。）

---

## 这是什么 / 给谁用

做研究的人硬盘里都有一个「下载文件夹地狱」：从知网、arXiv、SSRN、期刊官网下的论文 PDF，
文件名要么是一串编号（`1-s2.0-S0000000.pdf`），要么是 `download (3).pdf`，
过俩月根本不知道哪个是哪个。

**PaperTidy 专治这个。** 它读每个 PDF 里的**元数据（metadata，就是 PDF 内部记录的"这篇论文是什么"的信息）**，
自动查出这篇论文的作者、年份、标题，把文件名改整齐。

适合：社科 / 人文 / 理工科的硕博、科研人员、写文献综述的人——
凡是「文件夹里躺着几十上百个乱名 PDF、想一次性归档整齐」的场景。
**不需要你会写代码。**

## 用法①：双击运行（最省事，首推，不用装 Python）🖱️

1. 到本仓库 **Releases** 页面下载 `PaperTidy.exe`（单文件，约 44 MB，无需安装任何环境）。
2. **双击 `PaperTidy.exe`**，会弹出一个黑色小窗口，提示：
   > 把要整理的文件夹拖到本窗口再按回车, 或直接粘贴路径:
3. 打开你放论文的文件夹，把这个**文件夹**直接拖进那个黑窗口（或手动粘贴路径），按回车。
4. 它会先**列出预览**：打算把哪个文件改成什么名字，此时**一个文件都还没动**。
5. 看着没问题，**直接按回车**就真改；不想改就输入 `n` 再回车，它会「一个文件都没动」地退出。
6. 结束后窗口会停在「按回车键退出……」，看完结果再按回车关掉——**不会一闪而过**。

> 全程「先预览、你点头、才动手」，改错的风险几乎为零。

---

## 用法②：把文件夹拖到 exe 图标上

如果懒得双击再拖窗口，可以**直接把要整理的文件夹拖到 `PaperTidy.exe` 的图标上松手**，
效果和用法①一样：先弹预览 → 你确认 → 才改名 → 结束暂停。

## ⚠️ 首次运行会弹蓝色「Windows 已保护你的电脑」——这是正常的

因为这个 exe 是个人做的、**没有花钱买微软代码签名证书**，Windows 的 SmartScreen 第一次
遇到它会谨慎地弹一个蓝色窗口「Windows 已保护你的电脑 / Windows protected your PC」。

**这不是病毒报警，是"没签名"提醒。** 处理办法：

1. 点窗口里的「**更多信息 / More info**」（一开始是折叠的）。
2. 出现「**仍要运行 / Run anyway**」按钮，点它即可。

之后再运行就不会再弹了。（介意的话可以用下面「从源码运行」的方式，自己看代码更放心。）

## 用法③：命令行运行（会用命令行的人）

打开 PowerShell 或 cmd，把 exe 拖进去，后面跟上你要整理的文件夹：

```bat
:: 第一步：先预览，只看它打算怎么改，一个文件都不动（强烈建议先跑这个）
PaperTidy.exe "C:\我的\论文文件夹" --dry-run

:: 第二步：看着没问题了，去掉 --dry-run，真正改名（会先预览再问你确认）
PaperTidy.exe "C:\我的\论文文件夹"
```

- `--dry-run`：预览模式，只打印「打算把 A 改成 B」，不碰你的文件。
- `--yes` / `-y`：跳过确认直接改（写脚本、批处理时用，慎用）。
- `--no-pause`：结束不暂停（被别的脚本调用时用）。

> 提示：exe 第一次启动会慢几秒（它要把自己解压到临时目录），属正常现象。

## 用法④：从源码运行（会用 Python 的人）

```bash
pip install pdf2doi        # 唯一必需依赖，免费、无需 API key
python papertidy.py "你的文件夹" --dry-run   # 预览
python papertidy.py "你的文件夹"             # 真改
```

- Python 3.8+ 即可。
- 依赖见 `requirements.txt`（核心就是 `pdf2doi`）。

## 原理（为什么它不会把名字改错）

1. **提取身份证号**：用成熟开源库 [`pdf2doi`](https://github.com/MicheleCotrufo/pdf2doi) 从 PDF 里
   找出 **DOI**（数字对象唯一标识符，每篇正式发表论文的"身份证号"，形如 `10.1038/xxxxx`）
   或 **arXiv-id**（arXiv 预印本的编号，形如 `1706.03762`）。它会查 PDF 内嵌元数据、正文、文件名，多路兜底。
2. **查权威数据库**：拿到这个"身份证号"后，去 **Crossref / arXiv 官方 API** 查这篇论文
   **权威的**标题 / 作者 / 年份（免费、无需 key）。
3. **拼成规范文件名**：`第一作者姓_年份_标题.pdf`。

关键点：**标题永远来自官方数据库，不是从 PDF 首页乱抓的**——所以不会出现
「把首页的版权声明当成标题」这种低级错误。**一旦提不到可靠的 DOI/标题，它直接跳过，原文件一个字都不改。**

其它安全设计：文件名非法字符自动清洗、标题过长自动截断、重名自动加 `_2 _3`、
绝不把 identifier 写回你的原始 PDF。

---

## 注意事项

- **运行时需要联网**：因为要查 Crossref / arXiv 官方库。断网状态下大多数文件会被安全跳过。
- **务必先预览再确认**：双击/拖拽方式会自动先给你预览、等你点头；命令行养成先 `--dry-run` 的习惯，心里有数。
- **中文知网（CNKI）论文未充分测试**：本工具主要在英文 DOI / arXiv 论文上验证过；
  纯中文、无 DOI 的 PDF 可能提取不到信息而被跳过（跳过=不改，不会帮倒忙）。
- 只处理 `.pdf` 文件，不递归子文件夹（只改你指定的那一层）。
